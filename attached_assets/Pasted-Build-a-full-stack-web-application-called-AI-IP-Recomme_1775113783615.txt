Build a full-stack web application called **"AI IP Recommendation Copilot"**.

## 🎯 Objective

This application helps SAP sales and consulting teams identify the most relevant Intellectual Property (IP) solutions based on customer conversations.

The app should:

* Accept a meeting transcript (text or .vtt file)
* Analyze it using AI (LLM)
* Extract business problems, SAP context, and keywords
* Match against an IP catalog
* Recommend the **Top 3 most relevant IPs**
* Provide explanation + ready-to-use sales pitch

---

## 🧠 Key Concept

This is NOT just keyword matching.

This is a **mini RAG (Retrieval Augmented Generation) system**:

1. Extract insights from transcript
2. Match with structured IP catalog
3. Use LLM reasoning to recommend best IPs

---

## 🧠 AI LOGIC (IMPORTANT)

### Step 1: Extract Insights from Transcript

Call OpenAI with this prompt:

"Analyze the following customer meeting transcript and extract:

1. Key business problems
2. SAP modules or technologies mentioned
3. Business keywords
4. Industry (if possible)

Return ONLY valid JSON with keys:
businessProblems, keywords, sapModules, industry

Transcript:
{{transcript}}"

---

### Step 2: IP Matching Logic

Implement scoring logic (no need for embeddings in MVP):

Score each IP based on:

* Keyword overlap (40%)
* Business problem similarity (30%)
* SAP module match (20%)
* Industry match (10%)

Return top 3 IPs.

---

### Step 3: LLM Recommendation Reasoning

Call OpenAI again:

"Given the extracted insights and the following IP candidates, explain why each IP is relevant.

Return for each:

* reason (clear explanation)
* pitch (customer-ready message)"
